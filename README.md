# 🐍 Python Essentials for Data Engineering

A structured collection of Python fundamentals covering the core concepts every data engineer needs to know — from variables and loops to OOP, exception handling, and functional programming tools.

---

## 📂 Repository Structure

```
python-essentials-de/
├── variables.py            # Data types, assignment, type casting
├── conditionals.py         # if / elif / else, comparison & logical operators
├── loops.py                # for loops, while loops, break/continue
├── functions.py            # Defining functions, args, kwargs, return values
├── lists.py                # List creation, indexing, slicing, list methods
├── sets.py                 # Set operations, uniqueness, union/intersection
├── dictionary.py           # Key-value pairs, nested dicts, dict methods
├── string_methods.py       # String formatting, slicing, built-in methods
├── map,filter,reduce.py    # Functional programming with iterables
├── class,and,object.py     # OOP — classes, objects, constructors, methods
├── exception_handling.py   # try/except/finally, custom exceptions
└── request.py              # HTTP requests using the requests library
```

---

## 🧠 Topics Covered

| Topic | File | What's Inside |
|---|---|---|
| Variables | `variables.py` | Data types, type casting, naming conventions |
| Conditionals | `conditionals.py` | if/elif/else, comparison & logical operators |
| Loops | `loops.py` | for, while, break, continue, range() |
| Functions | `functions.py` | Defining functions, *args, **kwargs, return |
| Lists | `lists.py` | Indexing, slicing, append, sort, list comprehension |
| Sets | `sets.py` | Unique collections, union, intersection, difference |
| Dictionaries | `dictionary.py` | CRUD operations, nested dicts, .get(), .items() |
| String Methods | `string_methods.py` | strip, split, join, format, f-strings |
| Map/Filter/Reduce | `map,filter,reduce.py` | Functional approach to transforming iterables |
| OOP | `class,and,object.py` | Classes, __init__, methods, object instantiation |
| Exception Handling | `exception_handling.py` | try/except/finally, raising exceptions |
| HTTP Requests | `request.py` | GET/POST requests, JSON parsing, status codes |

---

## 🎯 Why These Topics Matter for Data Engineering

- **Data structures** (lists, sets, dicts) are foundational for transforming and processing datasets
- **Functions & OOP** enable writing reusable, modular pipeline code
- **Map/Filter/Reduce** are the basis of distributed computing models (think MapReduce, Spark)
- **Exception handling** is critical for building fault-tolerant ETL pipelines
- **Requests** enables API ingestion — a core pattern in modern data pipelines
